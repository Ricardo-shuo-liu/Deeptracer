# Deeptracer项目详细技术分工与任务清单

整体建立目标：

在GitHub提供一个库用于保存各个部分组件的信息，然后所有人通过git合作。

## 1. 后端负责人（1人）

### 技术栈要求
- **核心语言**: Python 3.8+

- **框架**: FastAPI, Click (CLI工具)

- **代码分析**: AST模块, Viztracer API等等

  {

  主要涉及，时间和空间线性图，以及python tutor源代码的部分提取-（注：tutor是web端的需要提取其中逻辑修改到本地）

  然后对于其他配件静态分析工具引入ApI，方便前端建立静态的工具箱。

  }

- **环境配置**: Docker基础, virtualenv

- **辅助工具**: Git, Pytest, logging模块

### 具体任务
- **第1周**:
  - [ ] 搭建Python项目骨架，配置virtualenv和依赖管理
  - [ ] 实现基础CLI命令 `deeptracer analyze [file]`
  - [ ] 同时实现API破坏性接口，import 后实现修改代码分析。
  - [ ] 设计安全沙箱执行环境（资源限制+超时控制）
  - [ ] 构建基础数据模型（代码结构、性能数据格式定义）
  
- **第2周**:
  - [ ] 集成Viztracer，实现性能数据捕获与解析
  - [ ] 实现AST解析器，提取代码结构信息
  - [ ] 设计并实现内部数据标准化格式
  - [ ] 开发FastAPI基础接口，支持前端数据请求

- **第3周**:
  - [ ] 实现数据预处理管道（清洗、转换、特征提取）
  - [ ] 开发重构建议应用机制（生成修改后代码）
  - [ ] 优化分析性能，添加进度反馈机制
  - [ ] 实现错误处理与日志记录系统

- **第4周**:
  - [ ] 集成所有后端组件，确保端到端流程
  - [ ] 与前端对接API，修复数据格式问题
  - [ ] 编写后端单元测试，覆盖核心功能
  - [ ] 生成技术文档，说明后端架构和接口规范

### 关键交付物
- 可工作的CLI工具
- 完整的代码分析管道
- 规范化的API接口文档
- 后端单元测试覆盖率报告

- **第2周**:

  - [ ] 完成Coze API集成代码，处理请求/响应
  - [ ] 开发数据转换层，将分析结果转换为AI可理解格式
  - [ ] 实现置信度评估机制（评估建议可靠性）

- **第3周**:

- AI提示模板库
- 建议置信度与影响评估系统

## 跨团队协作重点

1. **接口定义**（第1周）:
   - 后端与前端共同定义API规范
   - AI团队与后端确定数据格式
   - 可视化团队与前端确定组件接口

2. **集成点**（第2-3周）:
   - 后端→AI: 分析结果数据传递
   - AI→前端: 建议和解释数据传递
   - 后端→可视化: 原始分析数据提供
   - 前端→测试: 功能验收标准确认

3. **演示准备**（第4周）:
   - 全体参与核心场景演示流程设计
   - 测试负责人协调完整测试周期
   - 共同准备技术答辩材料
   - 演练故障恢复和备选方案

## 每日协作机制

- **晨会**（15分钟）: 同步进度、阻塞问题、当日计划
- **接口评审**（每周2次）: 确保模块间无缝对接
- **代码审查**（提PR时）: 关键模块必须2人审查
- **集成测试**（每周五）: 验证本周完成的集成点
- **文档同步**（持续）: 关键设计变更即时更新文档
